DOM

DOM (Document Object Model) : 웹 페이지의 구조화된 데이터 표현 (트리형태의 객체구조의 모델, 문서 객체 모델)

웹브라우저가 HTML 문서를 해석하고 화면에 해석된 결과를 보여주는 과정을 '렌더링'
1. 브라우저는 html 코드를 해석해서 요소들을 트리 형태로 표현하는 문서 'DOM'을 생성
2. DOM을 JS가 수정 ( 쉘 콘텐츠를 추가, 수정, 삭제하거나 마우스 클릭, 키보드 타이핑 등 이벤트에 대한 처리 )
3. DOM을 통해 화면에 웹 콘텐츠들을 렌더링

> HTML을 브라우저가 이해하고 조작할 수 있도록 만든 구조




document객체

DOM : 웹 페이지의 구조화된 트리 (HTML → 객체화된 구조)
window.document : 브라우저에 렌더링되는 HTML 문서 자체를 의미 (DOM 접근용 객체)
-> 페이지 콘텐츠, 즉 DOM에 대한 진입점 역할을 하는 프로그래밍 인터페이스

- 요소 선택 메소드
document.querySelector('p') -> 선택자를 인자로 전달받아, 문서 내 첫번째 요소를 반환
document.querySelector('#text')
document.querySelectorAll('css선택자')

document.getElementById('id값')
document.getElementsByClassName('class값')

- 속성 선택 메소드
const p = document.querySelector('p')
p -> 요소Element의 내용, 속성값들을 반환

p.textContent = "요소 내 텍스트 콘텍츠를 확인 / 변경 가능"
p.innerText <- textContent는 코드상 공백을 포함하지만 innerText는 텍스트만 가져온다
p.className
p.style( .color / .width )
p.title

p.innerHTML : 요소 내의 HTML코드를 텍스트 형태로 반환

- 코드 생성 메소드
document.write(x) : 문서에 콘텐츠 x를 추가 입력

document.createElement('div')
document.createElement('p')
-> 만들기만 하고 적용X


const p = document.createElement('p')
p.style.color = 'blue'

const div = document.getElementById('container')
div.appendChild(p)
-> append는 노드 객체와 문자열을 추가할 수 있고, appendChild는 노드 객체(html 요소, tag)만 추가할 수 있다.
-> append는 추가만 하고, appendChild는 반환도 한다는 특징도 있다.
